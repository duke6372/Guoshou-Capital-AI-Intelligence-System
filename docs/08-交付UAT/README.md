## UAT测试角色

| 角色 | 职责 |
|------|------|
| 客户业务人员 | 执行业务场景测试，验证功能满足业务需求 |
| 客户数据Owner | 验证数据口径和指标正确性 |
| 客户风控人员 | 验证风险预警规则和处置流程 |
| 实施团队 | 提供测试支持，协助问题定位 |
| 项目经理 | 协调测试资源，跟踪问题闭环 |

## 测试前提条件

1. **系统已部署**: 系统已在UAT环境部署完成
2. **数据已准备**: 测试数据已准备并脱敏
3. **账号已分配**: 测试账号已创建并分配权限
4. **培训已完成**: 相关人员已完成系统操作培训

## UAT测试流程

```
1. 测试准备
   ├── 阅读操作手册
   ├── 了解系统功能
   └── 准备测试数据

2. 执行测试
   ├── 按业务场景执行用例
   ├── 记录测试结果
   └── 截图/录屏留证

3. 问题反馈
   ├── 在Jira中提交问题
   ├── 标注问题等级
   └── 跟踪修复进度

4. 测试验收
   ├── 确认问题已修复
   ├── 回归测试
   └── 签署验收确认
```

## 测试通过标准

| 优先级 | 通过率要求 |
|--------|------------|
| P0用例 | 100%通过 |
| P1用例 | ≥95%通过 |
| P2用例 | ≥80%通过 |

**注**: P0用例为业务核心功能，必须全部通过方可验收。

## 问题等级定义

| 等级 | 定义 | 处理要求 |
|------|------|----------|
| 严重 | 功能不可用、数据错误、安全问题 | 24h内修复 |
| 一般 | 功能偏离、交互问题 | 3天内修复 |
| 轻微 | UI细节、体验优化 | 下一版本修复 |
