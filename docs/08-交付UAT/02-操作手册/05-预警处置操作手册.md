# 风险预警处置手册

## 概述

本手册指导风控人员处理风险预警，包括确认、指派、处置和关闭预警。

## 预警级别说明

| 级别 | 含义 | 处理时限 | 推送方式 |
|------|------|----------|----------|
| 红色 | 紧急需立即处理 | SLA 2小时 | 短信+系统+邮件 |
| 黄色 | 重要需关注 | 24小时 | 系统+邮件 |
| 蓝色/提示 | 一般提醒 | 下一工作日 | 系统 |

---

## 处置流程

```
发现预警 → 确认预警 → 指派处置 → 执行处置 → 关闭预警
    ↓
   升级（如无法处置）
```

---

## 步骤详解

### 1. 发现预警

**入口**:
- 工作台风险预警卡片
- 风险预警模块

**观察内容**:
- 预警级别（红/黄/蓝）
- 预警类型（出租率/能耗/租约到期等）
- 关联项目
- 触发条件
- 持续时间

### 2. 确认预警

**操作**:
1. 点击预警条目
2. 查看预警详情
3. 判断是否为真实风险
4. 点击"确认"按钮
5. 选择确认原因：
   - 确认异常，需要关注
   - 确认正常，数据波动
   - 确认为误报

**注意**: 确认操作是预警处置的第一步，也是责任认定的开始。

### 3. 指派处置

**操作**:
1. 确认后的预警点击"指派"
2. 选择责任人（从部门/人员列表选择）
3. 可填写指派说明
4. 提交

**指派原则**:
- 谁负责谁处置：选择项目负责人或处置人
- 及时指派：红色预警应在30分钟内指派
- 明确责任：指派时说明期望的处理方式

### 4. 执行处置

责任人收到指派后：

1. 查看预警详情和背景
2. 分析问题原因
3. 制定处置方案
4. 执行处置行动
5. 在系统中记录处置过程

**常见处置方式**:
- 出租率低：与租户谈判、调整租金、改进营销
- 能耗上升：检查设备、优化运营、调整策略
- 租约到期：启动续约谈判、准备招租

### 5. 关闭预警

**条件**: 问题已解决或风险已消除

**操作**:
1. 点击"关闭"
2. 选择关闭原因：
   - 问题已解决
   - 风险已转移
   - 确认为误报
3. 填写解决方案说明
4. 提交

**关闭标准**:
- 指标恢复到阈值以上
- 已制定并执行处置方案
- 或经评估确认无需处置

## 预警详情页面

### 基本信息

| 字段 | 内容 |
|------|------|
| 预警编号 | 系统生成的唯一标识 |
| 预警级别 | 红/黄/蓝 |
| 预警类型 | 出租率/能耗/租约到期等 |
| 关联项目 | 项目名称 |
| 触发时间 | 首次触发时间 |
| 持续天数 | 截至当前的天数 |

### 触发条件

| 字段 | 内容 |
|------|------|
| 当前值 | 实际指标值 |
| 阈值 | 触发预警的阈值 |
| 超出幅度 | 当前值与阈值的差距 |

### 证据数据

- 趋势图：指标变化趋势
- 对比数据：与历史同期/同类项目对比
- 相关文档：支持判断的相关资料

### 操作记录

显示预警的全生命周期：
- 创建 → 确认 → 指派 → 关闭
- 每次操作的时间、人、操作说明

## 批量操作

### 批量指派

当多个预警需要指派给同一人时：

1. 勾选多个预警（最多10个）
2. 点击"批量指派"
3. 选择责任人
4. 确认

### 批量关闭

当多个预警需要统一关闭时：

1. 勾选多个已处置预警
2. 点击"批量关闭"
3. 选择关闭原因
4. 确认

---

## 升级处理

### 升级条件

- 预警持续超过SLA时限未处置
- 处置遇到阻碍需要上级协调
- 风险扩大需要升级处理

### 升级操作

1. 点击"升级"
2. 选择升级原因
3. 选择升级对象（上级领导/专项小组）
4. 提交

### 升级影响

- 升级后相关人员收到通知
- 预警级别可能调整
- 生成升级记录

## 规则配置

管理员可配置预警规则：

### 查看规则

1. 进入"规则与推送"面板
2. 查看已有规则

### 规则要素

| 要素 | 说明 |
|------|------|
| 规则名称 | 如"出租率阈值规则" |
| 触发条件 | 指标+阈值+持续时间 |
| 豁免条件 | 豁免的项目/情况 |
| 升级策略 | 触发后的升级路径 |

### 规则变更

如需调整规则，联系系统管理员。

## 处置案例

### 案例1: 出租率预警

**场景**: 上海某项目出租率连续14天低于90%

**处置流程**:
1. **发现**: 系统中出现红色预警
2. **确认**: 确认是真实风险，与运营团队核实
3. **指派**: 指派给资产管理负责人
4. **处置**: 分析原因（主要租户退租），制定招租方案
5. **关闭**: 出租率恢复到90%以上后关闭

### 案例2: 能耗异常预警

**场景**: 北京某项目能耗成本同比上升18%

**处置过程**:
1. **确认**: 联系物业了解情况（空调系统更换）
2. **指派**: 指派给工程负责人
3. **处置**: 分析能耗构成，调整空调策略
4. **结果**: 能耗逐步下降，3个月后关闭预警

---

## 注意事项

1. **及时处理**: 红色预警应在2小时内响应
2. **操作留痕**: 每步操作都要记录，便于追溯
3. **证据保存**: 保留处置过程中的关键证据
4. **经验总结**: 处置完成后可沉淀到知识库

## 常见问题

### Q: 预警误报怎么办？
A: 确认时选择"确认为误报"，填写说明，系统会记录误报率用于规则优化。

### Q: 不是我负责的项目预警怎么办？
A: 可以"确认"但"指派"给正确的负责人，不要擅自关闭。

### Q: 预警无法关闭怎么办？
A: 检查指标是否真正恢复，或联系管理员排查问题。

### Q: 规则太严格/太宽松怎么办？
A: 联系风控管理员评估规则调整。
