# 合同知识调研报告（要点卡）

## 这份材料在解决什么问题（目的）
- 定义地产投资全生命周期中的合同知识结构（合同类型×关键条款×知识板块），评估AI在条款抽取、风险识别、条款比对、智能问答上的可行性，为KnowledgeBase模块的落地选型与范围提供依据。

## 覆盖的业务域/对象（范围）
- 覆盖8类核心合同：土地使用权、建设工程、合资/合作开发、租赁、物业管理、融资贷款、股权转让、资产证券化/REITs。[合同知识调研报告.pdf.md:L24-L116](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L24-L116)
- 归纳5大知识板块：合规、财务、运营、退出、风控。[合同知识调研报告.pdf.md:L117-L146](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L117-L146)

## 关键结论/事实（可追溯）
- 合资/合作开发合同被标注为“国寿资本最核心合同类型之一”，关键条款集中在股权/治理/表决/利润分配/优先购买权/退出机制等。[合同知识调研报告.pdf.md:L48-L58](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L48-L58)
- AI能力评估矩阵：多类合同在“知识提取/风险识别/条款比对/问答”上可行性较高，资产证券化类相对复杂。[合同知识调研报告.pdf.md:L148-L156](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L148-L156)
- 建议优先落地：租赁合同与融资贷款合同知识库（标准化高、结构清晰）；合资开发合同建议做专题知识图谱。[合同知识调研报告.pdf.md:L158-L161](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L158-L161)

## 明确的需求/能力点（用户故事/功能点）
- 合同解析：上传/导入合同→抽取关键字段→条款定位→风险点标注→输出审查结论与建议。[合同知识调研报告.pdf.md:L147-L165](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L147-L165)
- 条款比对：新合同 vs 历史同类合同（差异点、偏离惯例、潜在风险）。[合同知识调研报告.pdf.md:L169-L171](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L167-L171)
- 合同问答：围绕ROFR/Tag-along/退出机制/违约责任等高频问题，提供RAG式可追溯回答。[合同知识调研报告.pdf.md:L160-L165](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L160-L165)

## 数据与系统边界（数据源、上下游、依赖）
- 上游：合同管理系统/法务系统/项目档案；下游：风控预警、尽调清单、投审会材料与报告生成。

## 约束与风险（合规、权限、口径、时效）
- 合同属于高度敏感数据：必须落在严格的权限隔离、脱敏与审计留痕体系内，且需要“人工复核”闭环保证法律风险可控。

## 未决问题（需要向业务/技术确认）
- 8类合同的“样本文档覆盖率”与电子化质量：是否大量扫描件、是否存在版本/附件分散问题。
- 风险识别规则与输出形态：哪些风险可以“提示”，哪些必须“阻断/强制审批”，责任归属如何定义。
