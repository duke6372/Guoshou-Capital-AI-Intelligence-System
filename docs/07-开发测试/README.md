## 测试分层

| 层级 | 覆盖范围 | 自动化要求 | 触发时机 |
|------|----------|------------|----------|
| 单元测试 | 函数/组件逻辑 | 必须 | PR时 |
| 集成测试 | 模块间接口 | 必须 | PR时 |
| E2E测试 | 核心用户旅程 | 推荐 | 发布前 |
| AI评估 | 模型输出质量 | 必须 | 发布前 |

## 测试用例编写原则

1. **可执行性** - 每条用例需明确输入、预期输出、执行步骤
2. **独立性** - 用例间无依赖，可并行执行
3. **可重复** - 相同环境下结果一致
4. **可追溯** - 关联需求/设计文档

## 质量门槛

- 单元测试覆盖率 ≥ 80%
- P0用例通过率 100%
- P1用例通过率 ≥ 95%
- AI意图识别准确率 ≥ 90%
- 页面响应时间 P95 ≤ 3s

## 缺陷等级定义

| 等级 | 定义 | 修复时限 |
|------|------|----------|
| S0 | 系统崩溃、数据丢失、安全漏洞 | 24h |
| S1 | 核心功能不可用 | 48h |
| S2 | 功能偏离但不影响主流程 | 当前迭代 |
| S3 | UI/文案/体验问题 | 下一迭代 |
