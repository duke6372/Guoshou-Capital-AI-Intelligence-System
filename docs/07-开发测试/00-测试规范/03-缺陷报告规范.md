# 缺陷报告规范

## 1. 缺陷等级定义

| 等级 | 定义 | 响应时间 | 修复时限 | 发布阻塞 |
|------|------|----------|----------|----------|
| S0 | 系统崩溃、数据丢失、安全漏洞 | 1h | 24h | 是 |
| S1 | 核心功能不可用 | 4h | 48h | 是 |
| S2 | 功能偏离但不影响主流程 | 1d | 当前迭代 | 否 |
| S3 | UI/文案/体验问题 | 2d | 下一迭代 | 否 |

## 2. 缺陷报告字段

| 字段 | 说明 | 必填 |
|------|------|------|
| 缺陷编号 | 自动生成，格式：`BUG-年-序号` | 是 |
| 缺陷标题 | 简洁描述问题 | 是 |
| 严重等级 | S0/S1/S2/S3 | 是 |
| 优先级 | P0/P1/P2/P3 | 是 |
| 发现版本 | 发现时版本号 | 是 |
| 所属模块 | 影响的功能模块 | 是 |
| 复现步骤 | 详细操作步骤 | 是 |
| 预期结果 | 期望行为 | 是 |
| 实际结果 | 实际行为 | 是 |
| 环境信息 | 浏览器/系统/网络 | 是 |
| 截图/日志 | 复现证据 | 是 |
| 指派给 | 责任人 | 是 |
| 状态 | 新建/确认/修复中/待验证/关闭/重新打开 | 是 |

## 3. 缺陷状态流转

```
新建 → 确认（确认是缺陷）
  ↓
确认 → 修复中（开始修复）
  ↓
修复中 → 待验证（提交修复）
  ↓
待验证 → 关闭（验证通过）
       ↘ 重新打开（验证失败）
```

## 5. 缺陷分类统计

| 类别 | 定义 | 示例 |
|------|------|------|
| 功能缺陷 | 功能未实现或错误 | 按钮点击无响应 |
| 界面缺陷 | UI与设计稿不符 | 颜色/字体/布局偏差 |
| 数据缺陷 | 数据计算/展示错误 | 数值保留2位小数 |
| 性能缺陷 | 响应时间超标 | 页面加载>5s |
| 安全缺陷 | 权限/注入/泄露 | 越权访问数据 |
| AI效果缺陷 | 模型输出质量不达标 | 意图识别错误 |
