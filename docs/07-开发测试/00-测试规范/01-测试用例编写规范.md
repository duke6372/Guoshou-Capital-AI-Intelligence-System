# 测试用例编写规范

## 1. 用例结构

每个测试用例应包含以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 用例ID | 是 | 格式：`模块-编号`（如 `WB-001`） |
| 用例标题 | 是 | 简洁描述测试点 |
|前置条件 | 是 | 执行前必须满足的条件 |
|测试步骤 | 是 | 逐条操作指令 |
|预期结果 | 是 | 每步的预期输出 |
|优先级 | 是 | P0/P1/P2/P3 |
|测试类型 | 是 | 功能/接口/AI评估/性能 |
|关联需求 | 否 | 追溯需求编号 |

## 2. 用例ID编码规则

```
模块前缀:
- WB  : 工作台 (WorkBench)
- BI  : 经营分析 (BI Dashboard)
- PF  : 项目组合 (Portfolio)
- INT : 情报中心 (Intelligence)
- KB  : 知识库 (Knowledge Base)
- RG  : 风险预警 (Risk Guard)
- CP  : Copilot
- ADM : 管理后台 (Admin)
- GL  : 全局功能 (Global)

编号: 001-999
```

## 3. 优先级定义

| 优先级 | 定义 | 覆盖率要求 |
|--------|------|------------|
| P0 | 核心功能，阻断发布 | 必须100%通过 |
| P1 | 重要功能，影响主要流程 | ≥95%通过 |
| P2 | 一般功能，允许缺陷存在 | ≥80%通过 |
| P3 | 辅助功能，可选实现 | 可跳过 |

## 4. 用例编写示例

```markdown
### WB-001: 工作台KPI卡片数据展示

**前置条件**:
- 用户已登录系统
- 数据源已同步最新数据

**测试步骤**:
1. 进入工作台页面
2. 观察KPI卡片区域

**预期结果**:
- 在管规模卡片显示正确数值（如 ¥1,700亿）
- 出租率卡片显示正确数值（如 92.4%）
- NOI卡片显示正确数值（如 ¥2.84亿）
- 风险预警卡片显示未关闭数量（如 10）
- 各卡片同比/环比变化值正确

**优先级**: P0
**测试类型**: 功能测试
```

## 5. AI评估用例特殊要求

AI相关用例需额外标注：

```markdown
### CP-001: Copilot意图识别-经营分析

**评估维度**: 意图分类准确率

**测试数据**:
- 正面样本: 100条经营分析相关问题
- 负面样本: 50条其他模块问题

**评估指标**:
- 准确率 ≥ 90%
- 精确率 ≥ 85%
- 召回率 ≥ 88%

**优先级**: P0
**测试类型**: AI评估
```

## 6. 接口测试用例要求

```markdown
### API-001: 获取工作台KPI数据

**接口**: `GET /api/v1/workbench/kpi`

**请求参数**:
```json
{
  "date": "2026-05-16"  // 可选，默认当天
}
```

**预期响应**:
```json
{
  "code": 200,
  "data": {
    "managedScale": 170000000000,
    "occupancyRate": 0.924,
    "noi": 284000000,
    "riskAlerts": 10
  }
}
```

**错误码**:
- 401: 未授权
- 403: 无权限
- 500: 服务异常

**优先级**: P0
**测试类型**: 接口测试
```
