## Slide 3

项目背景 — 为什么现在？

• 管理规模持续扩大：22只在管基金，逾1700亿元签约规模
• 投资组合复杂度上升：境内+境外、股权+债权、办公+物流+产业多元业态
• 三大核心痛点：数据分散 · 情报滞后 · 知识流失
• 竞品压力加剧：平安不动产等头部玩家已在数字化上加大投入
• 技术窗口已成熟：国产大模型成熟，AI应用门槛大幅降低
• 政策支持：保险资金运用数字化转型受到监管鼓励

## Slide 9

模块详解 — DataHub 数据中枢

• 核心价值：打破数据孤岛，建立统一数据底座
• 内部数据连接器：基金管理系统 / 财务系统 / 物业管理系统 / 合同系统 / OA
• 外部数据采集器：代理行数据（戴德梁行/仲量联行） / 企查查 / 政策大数据平台 / 公开新闻
• 数据标准化引擎：统一数据字典 / 口径标准化 / 数据质量评分（完整性/准确性/时效性）
• 数据血缘追踪：从原始数据到分析结果的全链路可追溯
• 技术选型：Apache Airflow + PostgreSQL/TimescaleDB + Great Expectations

## Slide 10

模块详解 — BIDashboard 经营仪表盘

• 高管驾驶舱（一页式）：资产组合全景 / 核心指标（出租率/租金/NOI）/ 重大事项 / 趋势图
• 项目组合视图：GIS地图 + 列表双视图 / 业态/地域/合资方筛选 / 横向排名
• 单项目详情页：运营指标 / 财务指标 / 市场对标 / 租户分析
• 关键价值：报表生成从3-5天 → 即时，效率提升90%+
• 技术选型：Metabase / DataEase（开源BI）+ ClickHouse（OLAP）

## Slide 11

模块详解 — IntelEngine 情报引擎

• 竞争情报：多渠道自动监控（新闻/公众号/LinkedIn/监管公示）/ 置信度标注 / 分类整理
• 政策情报：实时订阅 + AI 24小时内解读 + 影响评估报告 + 历史政策库
• 竞品对标：出租率/租金/NOI 自动横向对比 + 差异原因 AI 分析
• 智能报告：市场周报、竞品月报自动生成，替代人工汇总
• 技术选型：MiniMax/智谱GLM（国产LLM）+ Milvus/Qdrant（向量数据库）+ Firecrawl（采集）

## Slide 12

模块详解 — KnowledgeBase 知识库

• 合同知识库：8类合同标准化解析，关键条款自动标注（违约责任/续约优先权/退出机制）
• 制度知识库：内部治理制度 / 外部法规库 / 监管合规追踪
• 案例知识库：历史项目案例复盘 / 投资决策复盘 / 最佳实践提炼
• RAG 应用场景：智能问答 / 尽调清单自动生成 / 合同条款比对 / 报告辅助写作
• 技术选型：Neo4j（知识图谱）+ BGE/text2vec（国产嵌入）+ PyMuPDF（文档解析）

## Slide 13

模块详解 — RiskGuard 风险卫士

• 五级预警体系：L1战略(年/季) → L2市场(月/周) → L3竞争(周/实时) → L4运营(实时) → L5风险(实时监控)
• 出租率预警：低于目标5%=黄色 / 低于目标10%=红色 / 月度环比下降>3%=持续监控
• 租户信用风险：工商信息变化监控 / 大客户经营异常信号 / 逾期30天黄色 / 逾期60天红色
• 政策合规风险：证照到期90天预警 / 保险资金运用合规自动检查
• 市场系统性风险：城市租金指数环比连跌3个月 / 利率上行触发融资成本测算
• 推送机制：红色→短信+推送+邮件 / 黄色→推送+邮件 / 蓝色→App推送

• 多端部署：飞书机器人（即时推送）/ Web对话平台（深度分析）/ 移动App / 邮件摘要
