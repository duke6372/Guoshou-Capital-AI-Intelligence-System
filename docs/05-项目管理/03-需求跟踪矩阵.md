## 4. 优先级定义

| 优先级 | 说明 |
|--------|------|
| P0 | MVP必须交付，否则系统不可用/不可验收 |
| P1 | 核心体验需求，必须在对应Phase交付 |
| P2 | 增强功能，可在后续迭代补充 |

## 5. 状态说明

| 状态 | 含义 |
|------|------|
| 待确认 | 需求已定义，等待干系人签字确认 |
| 已确认 | 需求已签字确认 |
| 开发中 | 正在开发实现 |
| 已交付 | 功能已完成并通过测试 |
| 已变更 | 需求发生变更，需重新确认 |
| 已关闭 | 需求不再实施（需说明原因） |
