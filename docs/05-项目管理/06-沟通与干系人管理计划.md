# 沟通与干系人管理计划

## 1. 干系人分析

| 干系人 | 项目角色 | 关注点 | 参与程度 | 沟通策略 |
|--------|----------|--------|----------|----------|
| 客户高管层 | 项目发起人/决策者 | ROI、投资回报、风险 | 阶段评审 | 简洁、聚焦价值与风险 |
| 客户业务负责人（投资/资管/风控） | 需求提供者/验收者 | 功能是否解决痛点、易用性 | 迭代演示/需求评审 | 演示+反馈循环 |
| 客户IT/数据团队 | 技术配合方 | 架构匹配、数据安全、运维交接 | 技术评审/周会 | 技术细节对齐 |
| 客户合规/安全团队 | 合规把关人 | 等保、数据分级、审计 | 阶段评审 | 合规清单进度同步 |
| 实施方项目经理 | 项目管理 | 进度、质量、成本、风险 | 日常管理 | 全量信息透明 |
| 实施方开发团队 | 交付执行 | 需求明确、技术可行性 | 每日站会/迭代规划 | 明确任务、解阻塞 |
| 实施方AI团队 | AI能力交付 | 数据质量、评估标准 | 技术评审 | 技术分析与对齐 |

## 2. 沟通矩阵

| 沟通类型 | 内容 | 频率 | 参与人 | 渠道 | 负责人 |
|----------|------|------|--------|------|--------|
| 项目周报 | 进度、风险、下周计划 | 每周一 | 全部干系人 | 飞书文档 | PM |
| 项目例会 | 进度同步、问题协调 | 每周 | PM+客户方负责人 | 飞书会议 | PM |
| 迭代规划会 | 下个迭代任务分配 | 每2周 | 开发团队+PM | 线下/飞书 | PM |
| 迭代演示 | 增量功能展示 | 每2周 | 干系人+团队 | 在线演示 | 产品经理 |
| 每日站会 | 进度/阻塞/计划 | 每日15min | 开发团队 | 站会 | 开发TL |
| 技术评审 | 架构/方案评审 | 按需 | 技术人员 | 会议+文档 | 架构师 |
| 变更控制会 | 变更评审与决策 | 按需 | CCB成员 | 会议纪要 | PM |
| 阶段评审会 | Phase交付物验收 | Phase结束 | 关键决策人 | 正式汇报 | PM |
| 风险评审 | 风险登记册回顾 | 每2周 | PM+核心团队 | 会议 | PM |

## 3. 沟通渠道工具

| 用途 | 工具 | 说明 |
|------|------|------|
| 即时沟通 | 飞书 | 日常沟通、问题快速对齐 |
| 文档协作 | 飞书文档 / Confluence | 需求文档、设计方案、会议纪要 |
| 任务管理 | Jira | 需求、任务、缺陷跟踪 |
| 代码仓库 | GitLab | 代码版本管理、Code Review |
| 设计稿 | Figma | UI设计、交互原型 |
| 知识库 | Confluence | 项目知识沉淀、运维文档 |

## 4. 报告模板

### 4.1 周报结构

```
# 项目周报 [日期] - [日期]

## 本周进度
- [模块] 完成情况
- [模块] 完成情况

## 下周计划
- [模块] 计划工作

## 风险与问题
- 🟢 🟡 🔴 [风险编号] [说明] [状态]

## 需要客户方协助
- [事项] [期望时间]

## 里程碑状态
- Phase 0: 按计划 / 有风险 / 延期
```

### 4.2 问题升级模板

```
# 问题升级

## 问题描述
[清晰说明问题、影响范围、当前状态]

## 建议方案
[选项A / 选项B / 推荐的方案]

## 所需决策
[需要决策人拍板的具体问题]

## 期望响应时间
[时间节点]
```

## 5. 会议管理原则

- 所有会议**提前发送议程**
- 会议**结束后24h内**发出纪要（含决策事项和行动项）
- 定期例会**严格遵守时长**（站会15min、演示45min）
- 决策类会议**必须有结论**
