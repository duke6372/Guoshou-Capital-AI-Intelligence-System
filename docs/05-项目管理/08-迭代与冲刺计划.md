## 3. 冲刺执行规范

### 每日站会

- **时间**：每天固定时间，15分钟
- **内容**：昨天完成了什么 / 今天计划做什么 / 有什么阻塞
- **原则**：只同步状态，不做技术讨论（技术讨论会后单独约）

### 冲刺规划

- **输入**：产品待办列表（Product Backlog）、优先级排序
- **输出**：冲刺待办列表（Sprint Backlog）、任务拆分
- **原则**：按"用户故事"组织、每个故事有验收条件、估算用故事点或人天

### 冲刺演示

- **内容**：可运行的功能演示，而非PPT
- **参与人**：团队全体 + 客户方干系人
- **反馈**：演示后收集反馈，纳入下个迭代规划

### 冲刺回顾

- **目标**：团队内部改进
- **讨论**：做得好什么 / 需要改进什么 / 下一步行动
- **产出**：明确1-2个改进项，下个迭代落实

## 4. 团队容量规划
