# 质量管理计划

## 1. 质量目标

| 质量维度 | 验收指标 | 目标值 | 测量方式 |
|----------|----------|--------|----------|
| 数据准确性 | 核心指标数据与源系统一致率 | ≥99% | 自动化对账 |
| 数据时效性 | 数据刷新延迟 | ≤T+1（运营数据） | 监控看板 |
| AI引用命中率 | AI回答有引用来源的比例 | ≥95% | 日志分析 |
| 意图识别准确率 | 用户意图识别正确率 | ≥90% | 抽样评估 |
| 页面响应时间 | 页面加载/数据查询 | ≤3秒（P95） | APM监控 |
| 系统可用性 | 核心服务可用率 | ≥99.5% | 健康检查 |
| 审计完整性 | 关键操作留痕率 | 100% | 日志校验 |
| MVP用户体验 | 高管5分钟完成"看总览→看异常→看穿透→导出" | 可用性测试通过 | 用户测试 |

## 2. 质量活动

### 2.1 设计阶段

| 活动 | 参与角色 | 产出 |
|------|----------|------|
| 设计评审 | PM + 开发 + 测试 | 设计评审纪要 |
| 指标口径对齐 | 数据Owner + 产品 | 口径字典确认签字 |
| 技术方案评审 | 架构师 + 开发 | 技术方案文档 |
| AI效果评估方案对齐 | AI工程师 + 产品 | 评估数据集 + 评估标准 |

### 2.2 开发阶段

| 活动 | 频率 | 责任人 |
|------|------|--------|
| 代码Review | 每次提交 | 开发TL |
| 单元测试 | 每日 | 开发工程师 |
| 自动化回归测试 | 每日 | CI流水线 |
| 前后端集成测试 | 每迭代 | 测试工程师 |
| 数据质量校验 | 数据上线时 | 数据工程师 |

### 2.3 测试阶段

| 活动 | 说明 |
|------|------|
| 功能测试 | 覆盖全部P0/P1需求 |
| 集成测试 | 跨模块流程验证 |
| AI效果评估 | 基于标注数据集的准确性评测 |
| 性能测试 | 模拟真实用户并发场景 |
| 安全测试 | 权限越权、注入、数据泄露检查 |
| 兼容性测试 | 浏览器/移动端多端兼容 |
| UAT验收测试 | 客户方业务人员参与验证 |

## 3. 缺陷管理

### 3.1 缺陷等级

| 等级 | 定义 | 修复时限 |
|------|------|----------|
| S0 致命 | 系统崩溃/数据丢失/安全漏洞 | 24h内修复 |
| S1 严重 | 核心功能不可用 | 48h内修复 |
| S2 中等 | 功能偏离但不影响主流程 | 当前迭代内修复 |
| S3 轻微 | UI细节/文案/体验优化 | 可选迭代修复 |

### 3.2 缺陷流程

```
发现缺陷 → 提单(Jira) → 开发认领 → 修复 → 自测 → Code Review
    → 测试验证 → 关闭 / 重新打开
```

## 4. 验收标准

### 4.1 Phase验收门禁

- 该Phase所有P0需求测试通过
- 无S0/S1级别未关闭缺陷
- AI类需求通过预定义的评估数据集测试
- 安全合规检查通过
- 验收演示获得干系人签字确认

### 4.2 最终验收条件

- 全部Phase交付物验收通过
- 系统稳定运行≥2周无S0/S1故障
- 运维文档与交接完成
- 知识转移培训完成
