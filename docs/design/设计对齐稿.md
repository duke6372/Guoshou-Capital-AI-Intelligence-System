# 国寿资本AI智能体情报系统：设计对齐稿（文字版）

## 0. 定位与北极星目标
- 产品定位：面向国寿资本/中国人寿投资中高管理层的“AI驱动情报与决策支持平台”，统一承载经营分析、情报研判、知识复用与风险预警，并具备可追溯与可审计能力。[00-背景综合.md](file:///Users/mic/Documents/trae_projects/Guoshou/docs/analysis/00-%E8%83%8C%E6%99%AF%E7%BB%BC%E5%90%88.md)
- 北极星目标：让管理层与一线团队从“找数据/找材料/找人问经验”转为“随时可查、可比、可追溯、可预警、可复用”，从而提升经营分析效率与决策质量。[产品设计方案.pdf.md:L23-L30](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%9B%BD%E5%AF%BF%E8%B5%84%E6%9C%ACAI%E6%99%BA%E8%83%BD%E4%BD%93%E6%83%85%E6%8A%A5%E7%B3%BB%E7%BB%9F%E4%BA%A7%E5%93%81%E8%AE%BE%E8%AE%A1%E6%96%B9%E6%A1%88.pdf.md#L23-L30)

## 1. 目标用户与权限模型（最小可行）
### 1.1 角色
- 超级管理员：租户/组织配置、系统接入、权限与审计策略、指标口径字典Owner配置。
- 数据管理员（Data Owner/数据治理）：数据字典/口径、数据质量、主数据对齐、血缘追踪管理。
- 高管层：驾驶舱、全局预警、战略情报、关键报告审批与查看。
- 投资团队：项目/竞品/市场情报、尽调、报告生成、对标分析、机会识别。
- 资产管理团队：项目运营看板、异常跟踪、风险处置协同、运营报告。
- 风控合规/法务：预警规则与处置闭环、合规清单、合同审查/条款比对、政策追踪。
- 审计/内控（只读）：访问日志、导出记录、报告与结论溯源、模型使用留痕。

### 1.2 权限原则（需要产品与设计共同遵循）
- 最小权限：按“组织/项目/基金/数据域/文档库”分层授权。
- 强审计：关键行为必须留痕（检索、问答、引用、导出、分享、预警确认/关闭）。
- 脱敏默认：对外发通道（邮件/IM卡片）默认只发摘要与跳转链接，原文与敏感字段在系统内按权限查看。[审核评估报告-要点卡.md](file:///Users/mic/Documents/trae_projects/Guoshou/docs/analysis/%E5%AE%A1%E6%A0%B8%E8%AF%84%E4%BC%B0%E6%8A%A5%E5%91%8A-%E8%A6%81%E7%82%B9%E5%8D%A1.md)

## 2. 信息架构（IA）
### 2.1 一级导航（建议）
- 工作台
- 经营分析（BI）
- 情报中心
- 知识库
- 风险预警
- Copilot（对话）
- 管理后台（仅管理员可见）

### 2.2 全局能力
- 全局搜索：支持跨“项目/指标/情报/知识/合同/报告”检索，带权限过滤与可追溯引用。
- 订阅与推送：情报订阅、预警订阅、报告订阅（按角色/项目/主题）。
- 引用体系：任何AI结论必须可点击回溯到来源数据/文档片段（证据链）。

## 3. 核心对象模型（Object Model）
### 3.1 业务对象
- 资产组合：基金/组合/区域/业态维度聚合
- 项目：项目信息、运营主体、地域/业态、合资结构、关键里程碑、关联合同、关联指标
- 指标：指标名称、口径定义、计算公式、数据源、刷新频率、质量评分、血缘追踪
- 合同：合同类型（8类）、版本、关键条款、条款标签、风险标签、关联项目/主体
- 情报条目：类型（战略/市场/竞争/运营/风险）、时效等级（L1-L5）、来源与置信度、关联对象、推送策略
- 风险预警：规则/阈值、级别、状态（新建/已确认/处理中/已关闭）、处置记录、责任人
- 报告：模板、章节、引用清单、审批流、导出与分享记录

### 3.2 AI对象（用于设计交互与审计）
- 问答会话：问题、意图识别、工具调用记录、返回结论、引用证据、用户反馈
- 生成任务：报告生成/摘要生成/条款比对任务，状态机与产物留存

## 4. 关键用户旅程（文字版）
### 4.1 旅程A：高管“晨间5分钟掌握全局”
1) 打开工作台/驾驶舱 → 看到昨日/当日关键指标卡（出租率/租金/NOI/重大事项）  
2) 查看红色预警摘要 → 进入预警详情 → 看到阈值触发原因与证据  
3) 一键生成“经营摘要”（可选）→ 系统生成含引用的简报 → 仅系统内查看/审批后外发

### 4.2 旅程B：投资团队“竞品/政策快速研判→生成报告”
1) 在情报中心订阅某竞品/城市/业态 → 接收L3竞争情报推送  
2) 打开情报详情 → 置信度、来源、关联项目/竞品主体、时间线  
3) 进入Copilot提问“最近半年竞品项目获取情况与我们对标差异” → 生成对比结论 + 引用  
4) 选择模板生成“竞品月报/专项研判” → 编辑引用与图表 → 发起审批 → 导出

### 4.3 旅程C：法务/风控“合同审查与条款比对”
1) 上传合同/选择合同版本 → 系统解析条款并标注风险点  
2) 选择“对比历史同类合同” → 输出差异清单（偏离惯例/潜在风险）  
3) 形成审查结论（可配置为提示/阻断/需审批）→ 留痕并通知相关责任人

### 4.4 旅程D：资产管理“异常预警→处置闭环”
1) 收到出租率/逾期等预警 → 查看趋势与阈值  
2) 生成处置建议（可选）→ 分配责任人/设定截止时间  
3) 在预警处置记录中更新进展 → 关闭预警并复盘（沉淀为案例知识）

## 5. 关键页面清单与交互说明（文字版）
### 5.1 工作台
- 入口：登录后默认落地页
- 核心区块：我的关注（项目/主题/竞品）、待处理预警、订阅情报、近期报告、常用入口
- 状态：空态（未订阅）、加载态、权限受限态
- 关键动作：订阅管理、跳转到预警/情报详情、发起报告生成、打开Copilot并带入上下文

### 5.2 高管驾驶舱（D1）
- 核心区块：资产组合总览、核心指标卡、趋势图（12个月）、重大事项/预警摘要、Top异常项目
- 关键交互：指标卡点击→进入指标解释（口径/来源/刷新）与穿透到项目列表；预警摘要点击→进入预警列表（默认过滤红色）；一键导出仅导出“聚合指标+图表+引用清单”，不带敏感明细

### 5.3 项目组合视图（D2）
- 视图：GIS地图 + 列表双视图（地图可选，至少保留列表与筛选）
- 筛选：地域/业态/基金/合资方/风险等级/指标区间
- 关键交互：地图点选/列表点选→项目详情；横向排名→切换指标维度（出租率/IRR/NOI等）；对标（选中多个项目→对比视图，差异归因可AI辅助但必须可回溯到数据）

### 5.4 项目详情页（D3）
- 分页/Tab：运营（出租率、租约到期分布、租户结构、异常事件）；财务（收入/成本/NOI/EBITDA、现金流摘要）；对标（同城同业态竞品或历史区间对比）；合同（关联合同清单、关键条款摘要、审查结论）；风险（相关预警与处置记录）
- 关键交互：任一图表支持“查看口径/来源/刷新时间/质量评分”；证据链从结论→回到数据与原文片段（合同/情报）

### 5.5 情报中心（列表）
- 列表维度：类型（五类）、时效等级（L1-L5）、主题、来源、置信度、关联对象、时间
- 关键交互：快速过滤只看“我关注的竞品/城市/业态”；批量操作（标记重要、加入报告、订阅/取消订阅）；质量控制（置信度低标记“需复核”并进入复核队列）

### 5.6 情报详情页
- 核心信息：标题、摘要、来源与抓取信息、置信度、关联实体/项目、时间线、引用与分享
- 关键交互：影响评估（选择一个或多个项目/组合→生成“可能影响点”与证据链）；加入报告（选择报告/新建报告→插入引用并保留来源链接）

### 5.7 知识库首页
- 分类：制度/流程SOP、合同知识、案例复盘、行业知识、退出策略
- 关键交互：录入/导入（模板化字段+附件，支持审核发布与版本管理）；检索支持按对象（项目/合同类型/条款标签/主题）过滤

### 5.8 合同库与合同审查页
- 合同库：按合同类型（8类）与项目维度浏览；支持版本与关键条款摘要
- 审查页（流程）：上传/选择合同→解析→展示条款目录与风险标签→选择历史同类合同对比→输出差异清单→形成审查结论（提示/需审批/阻断）并可导出审查报告（带引用）

### 5.9 风险预警中心
- 列表：按级别（红/黄/蓝）、类型（出租率/租户/逾期/政策合规/市场风险等）、项目、状态
- 详情：阈值规则、触发证据（数据点/时间线）、影响范围、处置建议、处置记录与复盘
- 关键交互：确认/指派/升级/关闭都必须记录原因与责任人；推送策略为红色短信+推送+邮件、黄色推送+邮件、蓝色App推送（可配置）。[产品设计方案.pdf.md:L276-L281](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%9B%BD%E5%AF%BF%E8%B5%84%E6%9C%ACAI%E6%99%BA%E8%83%BD%E4%BD%93%E6%83%85%E6%8A%A5%E7%B3%BB%E7%BB%9F%E4%BA%A7%E5%93%81%E8%AE%BE%E8%AE%A1%E6%96%B9%E6%A1%88.pdf.md#L276-L281)

### 5.10 Copilot（对话入口）
- 入口：一级导航独立入口 + 各页面右下角浮层（带上下文）
- 四类Copilot能力：经营分析、情报研究、知识问答、风险预警。[产品设计方案.pdf.md:L301-L319](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%9B%BD%E5%AF%BF%E8%B5%84%E6%9C%ACAI%E6%99%BA%E8%83%BD%E4%BD%93%E6%83%85%E6%8A%A5%E7%B3%BB%E7%BB%9F%E4%BA%A7%E5%93%81%E8%AE%BE%E8%AE%A1%E6%96%B9%E6%A1%88.pdf.md#L301-L319)
- 关键交互：每次回答必须展示“引用来源（数据/文档片段）”与“可执行下一步”按钮；支持“生成报告草稿/加入报告/创建订阅/创建预警处置任务”；提供反馈闭环（有用/无用、纠错、补充证据）

## 6. MVP范围（建议用于Phase 0快速试点）
目标：4–6周完成最小可用验证，降低全面推广风险。[汇报材料.pptx.md:L190-L194](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%9B%BD%E5%AF%BF%E8%B5%84%E6%9C%ACAI%E6%99%BA%E8%83%BD%E4%BD%93%E6%83%85%E6%8A%A5%E7%B3%BB%E7%BB%9F%E6%B1%87%E6%8A%A5%E6%9D%90%E6%96%99.pptx.md#L190-L194)

### 6.1 MVP必须交付
- 数据接入：至少1–2个内部系统或可替代的标准化数据导入，跑通口径字典与质量评分雏形
- 页面：工作台、驾驶舱（D1）、项目组合视图（D2）、项目详情（D3基础版）
- 风险：至少1类可量化预警（例如出租率阈值）+ 推送到系统内通知
- 报告：一键导出驾驶舱摘要（系统内导出）

### 6.2 MVP可选交付（视合规与资源）
- 合同：优先支持租赁合同/融资贷款合同的解析与条款比对试点。[合同知识调研报告.pdf.md:L158-L160](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%90%88%E5%90%8C%E7%9F%A5%E8%AF%86%E8%B0%83%E7%A0%94%E6%8A%A5%E5%91%8A.pdf.md#L158-L160)
- Copilot：仅开放“经营分析Copilot”的受控问答（白名单问题域+强引用+日志留存）

## 7. 埋点与度量（建议最小集合）
- 使用：DAU/WAU、驾驶舱访问占比、关键页面到达率（D1→D2→D3）
- 效率：报表准备耗时（基线 vs 上线后）、导出次数、人均查询次数
- AI质量：引用命中率、无引用回答占比、用户反馈有用率、纠错率、意图识别准确率（目标≥90%作为验收项之一）。[审核评估报告.pdf.md:L104-L104](file:///Users/mic/Documents/trae_projects/Guoshou/docs/source/%E5%AE%A1%E6%A0%B8%E8%AF%84%E4%BC%B0%E6%8A%A5%E5%91%8A.pdf.md#L104-L104)
- 风险：预警确认时延、处置完成率、重复预警率、误报率

## 8. 验收口径（对齐到评审）
- 数据：核心指标口径可解释、刷新频率可见、数据血缘可追溯（至少从展示→来源系统/文件）
- 体验：高管能在5分钟内完成“看总览→看异常→看项目穿透→导出摘要”
- 合规：权限可控、导出/分享留痕、敏感信息脱敏策略生效
- 运营：订阅与推送链路闭环（订阅→推送→查看→反馈/处置→归档）
